# スキルとメモリ（日本語）

Hermes Agentの最大の特徴は、使うほどに賢くなることです。**スキル**は手続き的な知識を保存し、**メモリ**は事実とコンテキストを記憶します。

> **注意:** 完全な英語版は [Skills](./skills.md) と [Memory](./memory.md) をご覧ください。

---

## スキルシステム

スキルは、エージェントが特定のタスクを実行するための再利用可能な手順書です。

### スキルの種類

| 種類 | 場所 | 説明 |
|------|------|------|
| **バンドルスキル** | `skills/` | Hermesに同梱、40以上 |
| **オプショナルスキル** | `optional-skills/` | 公式だが個別インストール |
| **ユーザースキル** | `~/.hermes/skills/` | エージェントが自動作成 |
| **コミュニティスキル** | Skills Hub | サードパーティ製 |

### スキルの検索とインストール

```bash
# スキルを検索
hermes skills search kubernetes
hermes skills search react --source skills-sh

# スキルをインストール
hermes skills install openai/skills/k8s
hermes skills install official/security/1password

# インストール済みスキルを確認
hermes skills list
```

チャット内で：

```
/skills              # スキル一覧を表示
/research            # "research"スキルを使用
/<スキル名>           # 特定のスキルを起動
```

### 自動スキル作成

Hermesは複雑なタスクを完了すると、そのパターンを自動的にスキルとして保存します。次回同様のタスクを要求すると、保存されたスキルを活用してより効率的に作業します。

```
❯ このリポジトリのテスト結果をSlackに投稿して

# エージェントが初回実行時にスキルを作成
# 次回は保存されたスキルを使って即座に実行
```

### スキルの自己改善

スキルは使用されるたびに改善される可能性があります。エージェントが既存のスキルを使用して問題に遭遇した場合、スキルを自動的に更新して改善します。

## メモリシステム

メモリは、セッション間で永続する事実とコンテキストを保存します。

### メモリの仕組み

```text
~/.hermes/memories/
├── MEMORY.md    # エージェントが管理する主要メモリファイル
└── USER.md      # ユーザーに関する情報
```

### メモリの種類

- **MEMORY.md** — プロジェクト、設定、環境に関する事実
- **USER.md** — ユーザーの好み、スキルレベル、ワークスタイル

### メモリの使い方

エージェントは自然に情報を記憶します：

```
❯ このプロジェクトではPython 3.11を使っていて、テストはpytestで実行するよ

# エージェントが自動的にMEMORY.mdに記録
```

明示的に記憶を指示することもできます：

```
❯ 覚えておいて：デプロイはmainブランチへのpushで自動実行される

# エージェントがMEMORY.mdに保存
```

### メモリの定期的な整理

エージェントは定期的にメモリを整理するよう自らを促します。古い情報を更新し、矛盾する情報を解決します。

### メモリプロバイダー

| プロバイダー | 説明 |
|------------|------|
| `markdown`（デフォルト） | ローカルMarkdownファイル |
| `honcho` | [Honcho](https://github.com/plastic-labs/honcho)による弁証法的ユーザーモデリング |

設定：

```yaml
# ~/.hermes/config.yaml
memory:
  enabled: true
  provider: "markdown"    # または "honcho"
```

## セッション検索

過去のセッションをエージェントに聞くことができます：

```
❯ あのDockerネットワークの問題、どう解決したっけ？

# エージェントがFTS5全文検索で過去のセッションを検索
# 結果をLLMが要約して回答
```

これが `session_search` ツールの仕組みです。過去の会話から学んだことを呼び出せるため、同じ問題に二度悩むことがありません。

## ベストプラクティス

### スキル

1. **汎用的なスキルをインストール** — 自分のワークフローに合ったスキルを探す
2. **スキルは自動作成に任せる** — 複雑なタスクを繰り返すと自動的にスキルが作られる
3. **Skills Guardを確認** — サードパーティスキルはインストール前にセキュリティレポートを確認

### メモリ

1. **重要な情報は明示的に伝える** — 「覚えておいて」で確実に記録
2. **定期的にメモリを確認** — `~/.hermes/memories/MEMORY.md` を開いて確認
3. **矛盾があれば修正を指示** — 古い情報があれば「これはもう変わったよ」と伝える
